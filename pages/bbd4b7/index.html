基于github中action服务的自动化打包部署

# 通过FTP部署服务

如果你有 FTP 服务器，可使用 FTP-Deploy-Action (opens new window) 将 github 代码推送到服务器上。Actions 步骤参考 GitHub 说明。

on: push
name: 🚀 Deploy website on push
jobs:
  web-deploy:
    name: 🎉 Deploy
    runs-on: ubuntu-latest
    steps:
    - name: 🚚 Get latest code
      uses: actions/checkout@v3

    - name: 📂 Sync files
      uses: SamKirkland/FTP-Deploy-Action@4.3.0
      with:
        server: ${{ secrets.ftp_host }}
        username: ${{ secrets.ftp_username }}
        password: ${{ secrets.ftp_password }}
        port: ${{ secrets.ftp_port }} # 建议更改默认的 21 端口

新建 FTP 时，需在云服务商的安全组和服务器上开放 FTP 端口，并临时暂停宝塔系统加固等安全插件（新建 FTP 容易与安全插件冲突）。

如果出现 FTPError: 530 Login authentication failed，则说明 FTP 密码错误或账号不存在，需用 FileZilla 测试 FTP 的有效性

确认 FTP 无效后，检查 FTP 密码是否填写正确，是否只有大小写字母和数字。如果密码错误，则在 github secrets 重新 update 密钥

如果密码正确，则进入 /www/server/pure-ftpd/etc/pureftpd.passwd，检查是否有该 FTP 账户。没有 FTP 账户的话，暂停宝塔系统加固等安全插件后，重新新建 FTP。

如果出现 Error: Timeout (control socket)，则说明同步服务器超时，可进入 Actions 页面点击右侧按钮Re-run all jobs，重新进行部署

如果错误连续出现，可以尝试关闭防火墙，测试是否 GitHub 服务器被拉黑了。

# 通过SSH进行部署

使用Github Action实现项目自动打包部署的步骤如下：

# 配置Github Action

在项目仓库中创建.github/workflows目录，并在其中创建一个yaml文件，用于配置Github Action。 下面是一个示例yaml文件：

name: Deploy
on:
  push:
    branches:
      - main # 只有在main分支push代码时才会触发
jobs:
  deploy:
    runs-on: ubuntu-latest # 运行环境
    steps:
      - name: Checkout # 检出代码
        uses: actions/checkout@v2
      - name: Install dependencies # 安装依赖
        run: npm install
      - name: Build # 打包
        run: npm run build
      - name: Deploy # 部署
        uses: easingthemes/ssh-deploy@v2
        with:
          server: ${{ secrets.SERVER }} # 服务器地址
          username: ${{ secrets.USERNAME }} # 用户名
          password: ${{ secrets.PASSWORD }} # 密码
          local_path: ./dist # 本地打包路径
          server_path: /var/www/html # 服务器部署路径

在该yaml文件中，我们定义了一个名为Deploy的Github Action，当在main分支上进行push操作时，就会触发该Action。 该Action包含一个deploy任务，该任务会在Ubuntu环境下执行以下步骤：

  1. 检出代码；

  2. 安装依赖；

  3. 打包；

  4. 部署。

在部署步骤中，我们使用了一个第三方的Actioneasingthemes/ssh-deploy来将打包好的代码部署到服务器上。

  1. 配置服务器信息

为了让Github Action能够自动将代码部署到服务器上，我们需要在Github仓库中添加一些密钥，用于验证服务器身份。 具体步骤如下：

  1. 在服务器上生成一个SSH密钥：
ssh-keygen
1

# 将生成的公钥添加到Github仓库中的Deploy keys中：

在Github仓库中点击Settings，进入Deploy keys页面，点击Add deploy key，将生成的公钥复制到Key字段中，并勾选Allow write access选项，最后点击Add key保存。

# 在Github仓库中添加服务器信息：

在Github仓库中点击Settings，进入Secrets页面，点击New repository secret，添加以下三个密钥：

SERVER：服务器地址；

USERNAME：登录服务器的用户名；

PASSWORD：登录服务器的密码。 注意：为了确保密钥的安全性，建议将密码等敏感信息保存在Github仓库的Secrets中，并在yaml文件中使用$的方式引用。

现在，每当我们在main分支上进行push操作时，Github Action就会自动执行打包和部署操作，将最新的代码部署到服务器上。
